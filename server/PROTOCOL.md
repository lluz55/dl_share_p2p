# P2P File Share — Signaling Protocol (Phase 1)

This document specifies the WebSocket messaging protocol used by the Go signaling server and TypeScript frontend.

## Inbound Messages (Client -> Server)

All messages are sent as UTF-8 encoded JSON strings.

### 1. `join`
Sent by the client to request creation of a new room (as host) or to join an existing room (as guest).

- **Payload Format**:
  ```json
  {
    "type": "join",
    "room": "room-code-here"
  }
  ```
- **Fields**:
  - `room` (string, optional): If empty or omitted, the server will generate a new three-word room code, assign the client the `host` role, and register them. If a valid room code is provided, the server will assign the client the `guest` role and add them to that room.

### 2. `offer`
Sent by a client to forward a WebRTC Session Description Protocol (SDP) offer to a specific target peer.

- **Payload Format**:
  ```json
  {
    "type": "offer",
    "room": "room-code-here",
    "to": "target-peer-id",
    "sdp": <opaque-sdp-json-object>
  }
  ```
- **Fields**:
  - `room` (string, optional): If provided, must match the room the sender is currently registered in (verified by the server).
  - `to` (string, required): The target peer's ID (e.g. the host ID or a guest ID).
  - `sdp` (object, required): The opaque SDP payload.

### 3. `answer`
Sent by a client to forward a WebRTC SDP answer to a specific target peer.

- **Payload Format**:
  ```json
  {
    "type": "answer",
    "room": "room-code-here",
    "to": "target-peer-id",
    "sdp": <opaque-sdp-json-object>
  }
  ```

### 4. `ice`
Sent by a client to forward a WebRTC ICE candidate to a specific target peer.

- **Payload Format**:
  ```json
  {
    "type": "ice",
    "room": "room-code-here",
    "to": "target-peer-id",
    "candidate": <opaque-ice-candidate-json-object>
  }
  ```

## Outbound Messages (Server -> Client)

All events are pushed by the server as JSON strings.

### 1. `joined`
Sent as an acknowledgment to the client that successfully executed a `join` message.

- **Payload Format**:
  ```json
  {
    "type": "joined",
    "room": "tiger-harbor-velvet",
    "self": "your-peer-id",
    "role": "host",
    "peers": ["other-peer-id-1", "other-peer-id-2"]
  }
  ```
- **Fields**:
  - `room` (string): The actual three-word room code.
  - `self` (string): The client's unique connection peer ID generated by the server.
  - `role` (string): `"host"` or `"guest"`.
  - `peers` (array of strings): The list of other peer IDs currently in the room.

### 2. `peer-joined`
Sent to all existing room participants when a new peer joins the room.

- **Payload Format**:
  ```json
  {
    "type": "peer-joined",
    "id": "new-peer-id"
  }
  ```

### 3. `peer-left`
Sent to all remaining room participants when a peer disconnects or leaves.

- **Payload Format**:
  ```json
  {
    "type": "peer-left",
    "id": "departed-peer-id"
  }
  ```

### 4. `error`
Sent when a request or message fails validation, or when a limit/restriction is violated. After sending an `error`, the server will close the WebSocket connection.

- **Payload Format**:
  ```json
  {
    "type": "error",
    "reason": "error-description-string"
  }
  ```
